FIXED Sitemap XML + Hreflang — ahora existe, bien estructurado

60 URLs con hreflang annotations, referenciado desde robots.txt. Bien.

CRITICAL Sitemap canonical domain mismatch

Todas las URLs del sitemap usan https://autoacg.com/en (sin www), pero el canonical de cada página usa https://www.autoacg.com/en. Esto envía señales contradictorias a Google. El dominio sin www redirige (307) a www, así que Google seguirá el redirect, pero es un hop innecesario que diluye señales canónicas en las 60 URLs indexadas.

→ Fix inmediato: Cambiar todos los <loc> del sitemap a https://www.autoacg.com/.... Actualizar robots.txt: Sitemap: https://www.autoacg.com/sitemap.xml. Resubmitir en GSC.

CRITICAL 307 Temporary Redirect — regresión desde 301 permanente

Cadena de redirect actual:

http://www.autoacg.com/  --308-->  https://www.autoacg.com/  --307-->  https://www.autoacg.com/en

Un 307 Temporary Redirect le dice a Google "este redirect es temporal, sigue indexando la URL original". Google puede mantener / en su índice junto con /en, creando contenido duplicado. Antes (Jun 5) esto era 301 (permanente). Es una regresión.

→ Fix inmediato: Cambiar a 308 Permanent Redirect en next.config.js. Si se debe mantener /en, el redirect debe ser permanente.

HIGH Hreflang: funcional pero frágil

Los hreflang están en HTTP Link headers (Google los soporta), pero NO en HTML <link> tags. Bing, Yandex y otros crawlers prefieren HTML. La implementación dual es best practice. x-default apunta a / que redirige a /en — funcional pero no ideal.

→ Fix: Añadir <link rel="alternate" hreflang="..."> en <head> como redundancia.

HIGH No <noscript> fallback

Con 34 script tags (13 external + 21 inline), si JavaScript falla, navegación, formularios y CTAs pueden romperse. Añadir un mensaje <noscript> es básico.

FIXED ¡Schema implementado! De 0 a 2 bloques JSON-LD

El cambio más grande de toda la auditoría. Ahora existen:

  • Bloque 2 (@graph, 20 nodos): Organization (con sameAs, ContactPoint), 4 Services con @id, BreadcrumbList, FAQPage (3 preguntas), 13 LocalBusiness con address + phone + hours

CRITICAL Sin coordenadas geo en los 13 LocalBusiness

Cada ubicación tiene calle, ciudad y teléfono pero latitud/longitud = MISSING. Sin geo, Google no puede confirmar ubicación física para Local Pack ni Google Maps. Este es el mayor bloqueante restante para visibilidad local.

→ Fix: Añadir "geo": {"@type": "GeoCoordinates", "latitude": 33.XXXX, "longitude": -118.XXXX} en cada LocalBusiness. Son 13 coordenadas que se buscan una vez y se hardcodean.

Desde Agosto 2023, Google restringió FAQ rich results a sitios gubernamentales y de salud. ACG no califica. Este schema nunca generará rich results. Es peso muerto y las 3 preguntas sin contenido visible constituyen riesgo de cloaking.

→ Fix: Eliminar FAQPage del schema. Si se quiere FAQ para AI/GEO, renderizar las preguntas y respuestas como HTML visible (sin FAQ schema).

HIGH AutoBodyShop en location pages es genérico (sin valor)

Las 13 páginas de ubicación (/locations/whittier, etc.) tienen el MISMO schema genérico: name="Auto Collision Group", phone="833-333-4224" (línea central), address completamente vacío, url siempre autoacg.com. Esto no aporta valor local.

→ Fix: Cada location page debe tener su propio LocalBusiness con: teléfono específico, dirección completa, geo, url de la página, y hasMap.

HIGH Faltan: aggregateRating, priceRange, hasMap, WebSite schema

Sin aggregateRating no hay estrellas en resultados. Sin WebSite schema no hay Sitelinks Searchbox. Sin priceRange y hasMap se pierde contexto para Local Pack.

MEDIUM BreadcrumbList — solo 1 ítem (inútil)

Un breadcrumb de un solo elemento ("Home") es ignorado por Google. Debe moverse a las páginas de ubicación con path completo: Home > Locations > CityName.
