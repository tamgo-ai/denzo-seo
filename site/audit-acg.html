2. Technical SEO — Score: 40/100

CRITICAL 237KB of Inline JavaScript — 62% of page weight
The page contains 237,479 bytes of inline JavaScript (62% of total HTML). This is Next.js framework code (13 JS bundles + inline chunks). While some is unavoidable with Next.js, this volume causes:
  • Slow Time to Interactive (TTI) — Google ranks based on Core Web Vitals
  • Poor mobile experience — 375KB download before rendering
  • Text-to-HTML ratio of only 3% — Google sees this as "thin content" despite 1,663 words
Impact: Core Web Vitals penalty, especially on mobile. Affects ALL keyword rankings.
→ Fix: Enable Next.js swcMinify: true. Use next/dynamic for below-fold components. Move non-critical JS to async or defer. Consider reducing JS bundle count.
HIGH Cache-Control: no-store — zero browser caching
Server returns Cache-Control: private, no-cache, no-store, max-age=0, must-revalidate. This means every visit downloads the full 375KB from scratch. For returning visitors and Googlebot, this adds unnecessary latency.
→ Fix: Set Cache-Control: public, max-age=3600, stale-while-revalidate=86400 for static pages. Use ISR (Incremental Static Regeneration) in Next.js for location pages.
HIGH URL /en suffix without hreflang tags
Homepage redirects to /en, implying an English version exists. But 0 hreflang tags are present. Google interprets /en as a separate URL from / — if you don't have other languages, this is unnecessary URL complexity.
→ Fix: Remove the /en redirect and serve directly from /. If multi-language IS planned, add proper hreflang tags: <link rel="alternate" hreflang="en" href="https://www.autoacg.com/en"> <link rel="alternate" hreflang="es" href="https://www.autoacg.com/es"> <link rel="alternate" hreflang="x-default" href="https://www.autoacg.com/">
PASS HTTPS + HSTS properly configured ✓
HTTPS enforced. HSTS max-age=63,072,000 (2 years). This is correctly configured and meets security best practices.

3. Schema & Structured Data — Score: 0/100

CRITICAL ZERO JSON-LD Schema Markup — completely absent
0 schema blocks found on the entire page. This is the single biggest SEO issue. Without schema markup, Google cannot:
  • Display rich results (star ratings, hours, FAQs, breadcrumbs)
  • Qualify for Google Local Pack and Google Maps integration
  • Show site links, business details, or service information in SERPs
  • Get cited by AI engines (ChatGPT, Perplexity, Google AI Overviews)
With 13 physical locations, this is a massive missed opportunity. Each location needs its own LocalBusiness schema block.
Impact: Invisible to rich results, local pack, and AI search. Estimated traffic loss: 30-50%.
→ Fix (Priority #1): Implement these schema types immediately:
